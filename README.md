# Hydro Optimize

HydroOJ 的定向功能优化插件。

## IOI 排名优化

对 `IOI` 赛制的排行榜应用以下规则：

1. 总分降序；
2. 总分相同则按达到该总分的用时升序；
3. 用时精确到秒；
4. 总分和用时都相同才显示相同名次；
5. 仍完全相同的选手按用户名的字典序升序显示。

“用时”显示在总分下方，表示从比赛开始至达到当前总分的 `HH:MM:SS`。它由组成该总分的正分提交中最晚的一次提交时间计算得出。

排行榜的总分后还会显示该选手的总提交次数，例如 `600 (17)`；每道已提交题目的得分后显示该题提交次数，例如 `80 (3)`。所有比赛内提交都会计数，不受评测结果影响。总次数为黑色，单题次数为灰色，提交次数不参与排名。

IOI 排行榜的单题成绩列会扩展为 `160px`，并允许横向滚动，以完整显示赛后双成绩、例如 `1000/1000 (12)`，避免提交次数被省略。

## 样例数据操作栏

题目页面的样例输入、输出数据会把“复制”和“填充到自测”操作移到标题同一行的独立操作栏中，不再覆盖长数据内容。输入数据保留“复制、填充到自测”，输出数据保留“复制”；原有按钮事件不变。

## 在线编程模式自动打开自测

在支持自测的题目中，点击“进入在线编程模式”时，插件会先记录自测面板应当可见，再等待 Hydro 的 Redux Store 和工具栏完成初始化；若面板尚未打开，会继续派发可见状态，直至输入、输出区域实际挂载。该功能不会运行代码、提交代码或覆盖已有自测输入。

## 在线编程页面布局

在线编程模式的操作区会调整为更清晰的布局：

1. “递交评测”固定在顶部工具栏最右侧；
2. “运行自测”放在自测输入框标题栏右端。输入、输出分栏由鼠标拖动改变大小时，该按钮会随输入框边界同步移动；
3. 输入、输出各自具有独立标题栏、边框与清晰的编辑区域；
4. 工具栏功能按钮使用黑色描边；“递交评测”使用黑色实底以突出主操作；
5. “退出”“递交评测”“运行自测”不再显示 `F9`、`F10`、`Alt+Q` 等快捷键文字或括号，但对应键盘快捷键仍然有效。

该布局只覆盖页面样式和按钮摆放，不替换 Hydro 的 React 评测、提交和退出事件；即使未安装 Hydro Front Beauty，也会使用内置的兼容颜色作为后备样式。

动态“运行自测”按钮挂载在 React 管理区域之外，并通过 `ResizeObserver` 跟随输入分栏；只有代理按钮成功显示后才隐藏工具栏中的原生按钮，因此面板加载失败时仍保留原生自测入口。布局会实时测量当前顶部导航的实际底边，并将工具栏设为不可收缩、纵向溢出可见，防止 Hydro Front Beauty 改变导航高度或拖动自测分栏重新布局时裁切顶部按钮。

布局脚本将组件挂载监听与工具栏状态监听隔离，并在更新工具栏期间暂停内部观察器，避免按钮文本、状态和样式写入递归触发自身，导致在线编程页面卡死或无法初始化。

## 安装

在 Hydro 服务器上克隆仓库后执行：

```bash
hydrooj addon add /root/.hydro/Hydro-Optimize
pm2 restart hydrooj
```

插件只会影响重启后生成的 IOI 排行榜视图；不会重写历史提交或比赛数据。

## 验证

创建一场 IOI 测试赛，使两名选手获得相同总分但在不同秒达到该分数。较早达到者应排在前面，且总分下方显示用时；若两人同一秒达到相同总分，则共享名次并按用户名排序。

进入支持自测的题目并点击“进入在线编程模式”：自测输入、输出区域应自动展开；“运行自测”应显示在输入标题栏右端并随分栏拖动；“递交评测”应位于顶部工具栏右端。关闭自测面板时，工具栏中的原生自测按钮应恢复显示。
