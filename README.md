# Hydro Optimize

HydroOJ 的定向功能优化插件。当前版本只优化 IOI 比赛排行榜，不修改 Hydro 核心源码、赛制数据或评测逻辑。

## IOI 排名优化

对 `IOI` 赛制的排行榜应用以下规则：

1. 总分降序；
2. 总分相同则按达到该总分的用时升序；
3. 用时精确到秒；
4. 总分和用时都相同才显示相同名次；
5. 仍完全相同的选手按用户名的字典序升序显示。

“用时”显示在总分下方，表示从比赛开始至达到当前总分的 `HH:MM:SS`。它由组成该总分的正分提交中最晚的一次提交时间计算得出。

排行榜的总分后还会显示该选手的总提交次数，例如 `600 (17)`；每道已提交题目的得分后显示该题提交次数，例如 `80 (3)`。所有比赛内提交都会计数，不受评测结果影响。总次数为黑色，单题次数为灰色，提交次数不参与排名。

IOI 排行榜的单题成绩列会扩展为 `160px`，并允许横向滚动，以完整显示赛后双成绩、例如 `1000/1000 (12)`，避免提交次数被省略。

## 样例数据操作栏

题目页面的样例输入、输出数据会把“复制”和“填充到自测”操作移到标题同一行的独立操作栏中，不再覆盖长数据内容。输入数据保留“复制、填充到自测”，输出数据保留“复制”；原有按钮事件不变。

## 在线编程模式自动打开自测

在支持自测的题目中，点击“进入在线编程模式”后，插件会在编辑器初始化完成时自动展开自测面板。该功能不会运行代码、提交代码或覆盖已有自测输入。

## 安装

在 Hydro 服务器上克隆仓库后执行：

```bash
hydrooj addon add /root/.hydro/Hydro-Optimize
pm2 restart hydrooj
```

插件只会影响重启后生成的 IOI 排行榜视图；不会重写历史提交或比赛数据。

## 验证

创建一场 IOI 测试赛，使两名选手获得相同总分但在不同秒达到该分数。较早达到者应排在前面，且总分下方显示用时；若两人同一秒达到相同总分，则共享名次并按用户名排序。
